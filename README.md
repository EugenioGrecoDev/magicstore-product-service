# MagicStore Product Service
A simple microservice to present CRUD methods and REST endpoints 

## Structure
**Intent:** understand the role of a microservice in the whole architecture  
**Languages:** JAVA SPRINGBOOT 4.0.3 JDK 21 MVN
**Dependencies:** JPA, H2, lombok, web, validation, security

\magicstore-product-service
- `MagicstoreProductSerivceApplication` – main
- \config
    - `SecurityConfig.java` – mockup login with admin/user and password/superpassword
- \controller
    - `MagicStoneController.java` – Get- Post- Put- Delete- Mapping with validation
- \model
    - `MagicSone.java` – database entity: id, serial number, color, size, name, description
    - `StoneColor.java` – enum list to have fixed options 
    - `StoneSize.java` – idem
- \repository
    - `MagicStoneRepository.java` – JPA and custom methods:  
        findBy -Color, -Size, -ColorAndSize, -SerialNumber, existsBySerialNumber
- \service
    - `MagicStoneService.java` – basic interface
    - `MagicStoneServiceImpl.java` – implementation of CRUD methods,  
                                    private method generateSerialNumber using color, size, random UUID
- \resources
    - `data.sql` – example data, db generated by hibernate

This service is built to be scalable and to fit naturally into a broader microservices architecture. It can operate on its own or be combined with an aggregator service to produce client‑facing DTOs. The serialNumber field uniquely identifies each product across external databases and other microservices, ensuring consistency. The security configuration separates user and admin privileges into read‑only and modify operations, anticipating future integration with a dedicated user‑service.
